# 1. DNS와 자원

## 도메인 네임과 네임 서버

IP 주소만으로 호스트 특정하는 것은 번거로움  
-> 일반적으로 사용자는 상대 호스트를 특정하기 위해 **도메인 네임** 사용

- **도메인 네임** : 호스트의 IP 주소와 대응되는 문자열 형태의 호스트 특정 정보
- **네임 서버** : 도메인 네임과 IP 주소 관리
  - 도메인 네임을 관리하는 네임 서버는 **DNS 서버**라고 부름 (Domain Name System)

 도메인 네임은 점(.)을 기준으로 계층적으로 분류

 전체 주소 도메인 네임 (FQDN) : 도메인 네임을 모두 포함하는 도메인 네임   
 FQDN 첫 번째 부분을 호스트 네임이라 부르기도 함.

## 자원을 식별하는 URI
**자원** : 네트워크 상의 메시지를 통해 주고받는 대상  
자원을 주고받기 위해서는 식별할 수 있어야 함

**URI** : 자원을 식별할 수 있는 정보
- **URL** : 위치 기반 식별자  
  <img width="661" height="83" alt="Image" src="https://github.com/user-attachments/assets/996336d3-dd7a-498d-884f-56567ac93276" />
  1. scheme : 자원에 접근하는 방법
  2. authority : 호스트를 특정할 수 있는 정보
  3. path : 자원이 위치한 경로. 슬래시를 기준으로 계층적으로 표현
  4. query : 추가적인 요청 파라미터 전달
  5. fragment : 자원의 한 조각을 가리키기 위한 정보
- **URN** : 이름 기반 식별자  
  URL은 자원의 위치가 변경되면 식별 불가, URN은 자원의 위치와 무관하게 식별 가능  
  하지만, 웹에서는 위치가 필요하기 때문에 실용성 낮음.
